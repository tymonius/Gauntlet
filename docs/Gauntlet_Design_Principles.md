# Gauntlet Design Principles

**Status:** Living design principles document  
**Purpose:** Record high-level design standards used to evaluate Gauntlet rules, factions, cards, victory conditions, and future variants.

These principles are not card text or release rules. They are design constraints for judging whether a proposed rule or system belongs in Gauntlet.

---

## 1. Run the Gauntlet

The player-facing identity of the game should center on **the Gauntlet**.

Gauntlet is about advancing through contested Territories, winning battles, occupying ground, surviving counterattacks, capturing Territories, and creating decisive breakthrough pressure.

Heartlands remain important rules endpoints and final defensive positions, but they should not become the main player-facing hook or the primary language used to describe the game.

---

## 2. Parallel Progress Principle

Both players should usually be able to make meaningful progress toward at least one victory condition at the same time.

Base Gauntlet can become a pure tug-of-war: when one player advances, the other player is often only losing ground. Faction victory conditions should reduce that problem by creating overlapping clocks and parallel forms of progress.

A faction victory condition should not merely confirm that a player is already winning the normal board-position race. It should let that faction make productive progress even when the opponent is also making productive progress.

The goal is for both players to feel late into the game that they are doing something meaningful and could still win.

---

## 3. Alternate Victories Should Be Gauntlet-Native

Alternate victory conditions should create parallel progress, but they should still be tied to Gauntlet events.

Good alternate victory progress should come from things like:

- movement,
- battles,
- occupation,
- capture,
- counterattacks,
- card commitments,
- battle draw,
- territory position,
- accepted or refused Terms,
- cards entering or leaving the Graveyard,
- visible faction resources,
- revealed or contestable board states.

Alternate victories should not become disconnected solitaire systems. They should create pressure the opponent can see, understand, race, disrupt, or answer through normal play.

---

## 4. Avoid Win-More Victory Conditions

A faction victory condition should not require that faction to already be succeeding at the normal conquest path unless conquest is that faction's actual identity.

If an alternate victory requires the player to already have superior board position, it risks becoming a win-more condition rather than a real alternative path.

The test is:

> Can this faction make progress toward its victory condition while the opponent is making progress toward theirs?

If not, the condition should be revised.

---

## 5. Factions Change the Path, Not the Game

Factions should change how players run the Gauntlet, not replace Gauntlet with unrelated minigames.

A faction should have a distinctive pressure pattern, resource, leader identity, and possible victory path, but it should still care about the shared core systems.

The more a faction asks players to track something new, the more that system must justify itself through interesting decisions, visible pressure, and meaningful counterplay.

---

## 6. Visible, Interactive Pressure

Faction progress should be visible enough that the opponent understands the danger and can respond.

Hidden information can exist, especially for Intelligence, but the overall pressure should not be unknowable or arbitrary. Players should be able to recognize when they are under threat and make informed decisions about racing, disrupting, retreating, accepting risk, or changing strategy.

---

## 7. Neutral Cards and Shared Counterplay

Before moving a neutral card into a faction, check whether that card provides an important shared tool other factions still need.

Faction identity matters, but not every thematically aligned card should become faction-locked. Some cards should remain neutral because they provide universal counterplay, emergency defense, pacing correction, comeback tools, or interaction with otherwise hard-to-answer strategies.

---

## 8. Exact Text Before Judgment

During card migration, card review should begin with exact current card text before any design judgment.

Do not infer a card's effect from its name. Do not summarize from memory when exact card text is available.

Card review should include:

1. current cost,
2. current classification,
3. exact Action text,
4. exact Battle text,
5. exact reminder text, if any,
6. candidate faction or neutral destination,
7. faction-mechanic compatibility check,
8. neutral-removal check when moving a card out of the neutral pool.

---

## 9. Complexity Must Earn Its Place

Gauntlet can support rich systems, but complexity should create decisions, tension, counterplay, or clarity.

Avoid adding layers only because they are elegant. Prefer systems that are easy to expose to players, easy to test, and hard to misunderstand.

When a system becomes too abstract, fiddly, or isolated from board pressure, simplify it or cut it.
